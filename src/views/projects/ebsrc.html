EBS 리딩클럽 B2C / B2B Design

[기여도] UX/UI 디자인 (90%)

iPortfolio에서 가장 기여를 많이 한 프로젝트로 자사 대표 서비스의 UX/UI 디자인 업무를 진행 했습니다. 국내 영어 교육에 대한 문제를 해결하고 자체 교육에 대한 수요를 충족시켜 줄 수 있는 솔루션을 제시합니다. EBS 리딩클럽에서 시작된 B2C/B2B 서비스를 시작으로 확장된 중국, 일본, 터키 국가에 글로벌 서비스의 시작을 함께 할 수 있었습니다.

[목표]

국내 영어 교육에 대한 문제를 해결하고 자체 교육에 대한 수요를 충족시켜 줄 수 있는 솔루션을 제시합니다.

THE PROJECT

EBS 미디어, 옥스퍼드 대학 출판부와 협력하여 탄생한 EBS 리딩클럽은 자체적으로 개발한 5단계 학습법을 제공합니다.
게임을 활용한 단어 학습을 시작으로 내용 가리고 상상하며 읽기, 보고 말하기 마지막은 스펠링 게임으로 마무리합니다.
Web Responsive, Android, iOS (Tablet, Mobile) 모든 환경에서 사용할 수 있으며 온라인에서든, 오프라인에서든 쉽고 반복적인
학습을 도와줍니다.

Service Flow

별도의 기획 포지션이 없었기에 사용성을 개선하기 위한 선행 작업으로 서비스 Flow를 작성하여 흐름을 명확히 하는 것부터 시작
하였습니다. SKETCH NESTED SYMBOL, OVERRIDE 컴포넌트 조합을 활용해 심벌의 낱개 형태가 아닌 최소한의 심벌로 최대한의
활용 가짓수를 적용한 LIBRARY 파일의 디자인 시스템을 만들었습니다. DESIGN SYSTEM은 WEB/APP 다양한 플랫폼의 UI를 일관적이고 확장 가능한 형태로 만들어 사용자 경험을 제공/개선하는 데 큰 도움이 되었습니다.

Service Concept

국내 교육 시장에서 신뢰를 받는 EBS 미디어, 100년 이상의 역사와 전통을 가진 옥스퍼드 대학 출판부의 이미지를 시각화하기 위해 시안 성이 강력한 그린 컬러와 블루 컬러를 포인트로, 서브 컬러는 무채색으로 배치해 파트너사의 아이덴티티를 표현하였습니다. 콘텐츠에 주목도를 높이기 위한 화면 배치에 중점을 두어 디자인 했습니다.

Design System

Web Responsive, App(iOS, Android) Tablet, Mobile 화면을 혼자 설계하고 디자인하기에는 시간적, 정신적인 여유가 부족했습니다. 서비스에서 제공하는 기능은 많았고 업무를 효율적으로 처리하기 위해 SKETCH NESTED SYMBOL, OVERRIDE 컴퍼넌트 조합을 활용해 디자인 시스템을 만들었습니다. 공통으로 사용되는 Symbol을 모아 Core Library 파일을 만들고 Typography, Color, Button 등
컴퍼넌트들의 목적과 기능별로 분류해 EBS 리딩클럽의 Design System을 만들었습니다.
그리고 디자인된 결과물과 개발 결과물의 차이를 줄이기 위해 개발 컨벤션 사이트의 구축을 제안했습니다.

Design System 제작 과정

디자인 시스템을 이용해 정해진 규칙을 쉽게 공유하고 재사용 가능하게 해 여러 명이 디자인하더라도 빠르고 일관되도록 하는 것이 목적이었습니다. Sketch Core Library 파일을 이용해 여러 명이 작업하더라도 빠르고 일관된 서비스 경험을 유지하도록 초기 설계 단계에서 많은 시간을 들였습니다. EBS 리딩클럽은 국내뿐 아니라 아시아 및 유럽 국가에 서비스를 확장해 사용하고 있기 때문입니다. 아래와 같은 순서로 작업을 진행했습니다. 처음부터 모든 컴포넌트를 완벽하게 작업할 수 없기에 완성된 부분부터 차례대로 공유해 피드백을 주고받으며 개발 적용 가능한 범위부터 도입하는 방식으로 진행했습니다.

UI/UX Design

자체적으로 개발한 MDR 학습법은 도서를 읽는 기능과 더불어 책 속에 있는 단어와 문장들로 스펠링/카드 게임, 듣기, 말하기 기능을
다양한 기기에서 사용할 수 있도록 디자인하였습니다.

Viewer Design

기능 개발의 시나리오 정립, Web/App(Tabelt, Mobile) UI/UX 디자인, 화면에 적용되는 인터렉션 작업을 100% 진행했습니다.
5단계 학습 기능의 시나리오, 디자인을 진행했고 사용성이나 콘텐츠의 개선 사항에 대한 부분을 피드백을 받아 단계별 학습 단계의 UI/UX 전체 구성 및 개선 작업을 진행했습니다.

"Warm Up" 단계 UX 개선 진행

프로토타입 및 모션 작업을 진행했고 Principle 툴을 활용했습니다. 내부 직원 및 학부모 체험단을 대상으로 유저 테스트를 진행했습니다.

문제점

추상적 의미의 단어는 그림 카드로 표현하기에 한계가 있습니다. 책 속 단어와 다른 의미의 그림카드가 들어가는 경우가 있습니다.

  • star → 별 모양, 별, 가수
  • cross → 화난, 십자가, 건너다

해결 방안

  • 모호한 그래픽 이미지 대신 실사 이미지를 사용
  • 추상적 단어에 예문 표시
  • 단어가 갖고 있는 의미의 이미지를 모두 제공, 여러 의미가 있다는 것을 확인하는 인지적 학습효과

모바일 Drag & Drop 제스쳐 개선

기존에 사용되는 레이아웃은 그림 카드를 표시하는 데 무리가 없었지만, 실사 이미지를 사용하며 몇 가지 이슈가 있었습니다. 요소가 많은 이미지의 경우 인식이 힘들고 4개 이상의 이미지 적용 시 터치 영역을 확보하는 데 어려움이 있었습니다. 그 과정에서 drag & drop 기능을 적용하는 방향으로 화면 진입 시 그림 카드가 랜덤하게 흩뿌려져 보이게 제안해 적용했습니다.

보상에 대한 심리적 애니메이션 개선

인터렉션에 관한 부분은 Principle이라는 프로토타이핑 툴을 이용해 실제 앱을 사용하는 것처럼 데모 앱을 제작했습니다. 사용성 테스트는 내부에서 진행했고 단어를 맞췄을 때 즐거움을 주기 위한 애니메이션의 경우 시안을 여러 개 만들었는데 시안 모두 호응이 좋아 하나의 모션이 아닌 저 레벨과 높은 레벨의 모션으로 구분해 적용할 수 있었습니다.

"Speck Up" 단계 UX 개선 진행

2016년 7월 첫 번째 배포 이후 2019년 7월 개선 작업이 진행되었습니다. 추가 기능 개발 건에 대해서는 시나리오, 인터렉션, UI/UX 디자인을 담당했습니다.

문제점

기존 기능은 듣고, 말하는 것을 녹음하는 정도라 엉망인 데이터로 녹음을 진행해도 다음 단계로 넘어갈 수 있고 사용자가 녹음 한 문장에 대한 피드백도 제공하지 않았습니다.

해결 방법

Language Confidence 서비스 API를 이용해 서비스에 접목시켰습니다. 발음을 분석해 단어 단위로 잘 된 발음과 부족한 발음을 확인할 수 있는 기능을 제공합니다. 사용자의 발음을 기억해 자주 헷갈리는 음소를 포함한 단어를 Drilling 하는 개인별 맞춤학습을 제공합니다. 사용자의 발음 오류를 파악 후 가이드라인을 제공해 발화에 익숙하지 않은 사용자에게 도움을 줄 수 있습니다.

난이도 표현 디자인

난이도의 쉽고 어려움을 표현하면서 구분이 차별점을 두지 않는 시각적인 요소를 표현하는 것에 집중했습니다. 또 해당 기능을 사용하지 않을 사용자를 고려해 분석 기능을
On/Off 할 수 있는 옵션을 추가했습니다.

Service flow

아래 포인트에 집중해 디자인했습니다.

  • 사용자가 녹음한 발음에 대한 결과를 상/중/하 단계로 시각적으로 표현할 것
  • 사용자가 진행한 모든 문장에 대한 결과(Good, Excellent)를 보여줄 것

기존에 만들어놓은 UI를 사용하는 것보다 개선된 기능에 맞추어 화면에 하나의 기능씩 잘 보이게 하는 방향으로 진행했습니다. 시선을 분산시키고 집중력을 저하시키는 거추장스러운 요소는 제거하고 점수 화면을 상단에 표시했습니다. 시각화할 수 있는 부분은 애매한 그래픽보다는 텍스트로 명확하게 보여줄 수 있도록 고려해 시안 작업을 진행했습니다.

Interaction Design

내부에서 개발된 API를 사용하는 것이 아니기에 사용자가 음성을 녹음하고 결과 값을 확인하는 데까지 로딩은 3초 이상 걸렸습니다. 기다림을 지루하지 않게 표현하기 위한 애니메이션이 필요했고 프로그레스 바, 슬롯머신, 오래된 타자기 모션을 진행했습니다. 모션이 표현되는 동안 사용자에게 안내할 수 있는 가이드 멘트를 하단에 추가해 학습에 도움을 주는 효과를 제공했습니다.
